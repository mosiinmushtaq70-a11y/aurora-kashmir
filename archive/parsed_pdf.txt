AURORA LENS
Global Aurora Prediction Platform — Project Specification
Prepared by: Mosin | B.Tech AIML, Year 1, Semester 2
Build period: 7 days | Stack: Python, XGBoost, Streamlit, NASA OMNI
Aurora Lens is a real-time, ML-powered aurora visibility forecasting platform for photographers, scientists
and aurora hunters worldwide. Given any location on Earth, it predicts whether the Northern (or Southern)
Lights will be visible in the next 3 hours — along with intensity, optimal viewing window, cloud cover, and a
live globe heatmap of aurora probability.
1. WHAT WE ARE BUILDING
Core Value Proposition
A photographer in Tromsø, a scientist in Helsinki, or an aurora enthusiast in Ushuaia can open Aurora
Lens, enter their location, and instantly know whether tonight is worth stepping outside. The platform
combines 60 years of historical space weather data with live NASA/NOAA feeds to produce predictions
that are competitive with professional operational forecasts.
Key Features
1. Live aurora intensity forecast for any lat/lon on Earth (both hemispheres)
2. 3-hour ahead prediction using a trained XGBoost two-stage ML model
3. Global heatmap showing aurora probability contours on an interactive globe
4. Weather integration — cloud cover, visibility and moon phase at the viewing spot
5. Push/email alerts when conditions cross the visibility threshold for saved locations
6. Historical storm archive — every major aurora event since 1995 with full context
7. Photographer tips — best camera settings, exposure times, location quality score
8. Solar wind dashboard — live Bz, Kp, solar flare feed, 3-day forecast chart

Note on Leaky Features
The OMNI file also contains ap, AE and Dst indices. These are geomagnetic activity indices measured
simultaneously with Kp — using them as features to predict Kp would be circular (data leakage). They are
dropped from all feature sets before training.

3. MACHINE LEARNING MODEL
Architecture — Two-Stage XGBoost
A single 5-class model struggles because 72% of all hours are quiet (Kp < 3). The two-stage design
separates the problem into two simpler questions:
Stage 1: Is anything happening? (binary: Kp < 3 vs Kp >= 3)
Stage 2: If yes, how intense? (4-class: Low / Moderate / Strong / Extreme)
If Stage 1 predicts Quiet, the final answer is None. Only if Stage 1 predicts Active does the input pass to
Stage 2. This mirrors how a real forecaster thinks and dramatically improves recall on rare extreme events.

Class Imbalance — SMOTE
The Extreme class (Kp >= 8) has only ~650 occurrences out of 270,000 rows. Without correction the
model ignores it. SMOTE (Synthetic Minority Oversampling Technique) generates synthetic Extreme
examples by interpolating between real ones, bringing the class ratio to a trainable balance. Combined
with compute_sample_weight('balanced'), this improves Extreme recall by 15-20%.

8. KEY SCIENTIFIC FACTS
Kp index ranges from 0 (completely quiet) to 9 (extreme geomagnetic storm)
Aurora becomes visible at mid-latitudes (40-50 degrees N/S) only when Kp reaches 5+
Kashmir sits at geographic latitude 34 degrees N — needs Kp 8+ for naked-eye aurora
Tromsoe, Norway at 70 degrees N sees aurora even at Kp 1
The Russell-McPherron effect means equinox months (March, April, September, October) produce
stronger storms
Solar Cycle 25 peaked in 2024-2025 — the most active aurora period in 20 years
Negative Bz (southward IMF) is the single strongest predictor of aurora — it opens Earth's
magnetosphere
CMEs travel from Sun to Earth in 15-72 hours depending on speed (400-3000 km/s)
The aurora oval expands equatorward as Kp increases — at Kp 9 it reaches 25 degrees latitude
