# Architecture

## Overview
AuroraLens is a full-stack web application designed to predict and visualize Aurora presence and probabilities. It combines a visually rich Next.js frontend with 3D elements (Three.js) and a high-performance Python FastAPI backend that runs Machine Learning models (XGBoost) for real-time space weather forecasting.

## Components

### Frontend (Next.js)
- **Purpose:** Provide an interactive, responsive UI with real-time 3D models and visually appealing animations.
- **Location:** `frontend/`
- **Dependencies:** React, Three.js, React Three Fiber, React Drei, GSAP, TailwindCSS.

### Backend API (FastAPI)
- **Purpose:** Serve real-time telemetry and ML-driven aurora predictions to the frontend.
- **Location:** `api/main.py` and `api/routers/`
- **Dependencies:** FastAPI, Uvicorn, Pandas.
- **Dependents:** Frontend.

### Space Weather & ML Engine (Python)
- **Purpose:** Fetch space weather data, process it, and run inferences using the pre-trained XGBoost model.
- **Location:** `src/space_weather.py`, `src/predictor.py`, `src/train_model.py`
- **Dependencies:** Pandas, XGBoost, Scikit-learn, Requests.
- **Dependents:** Backend API.

## Data Flow
1. **Telemetry Fetching:** The backend (`space_weather.py`) polling NOAA APIs for solar wind (DSCOVR), plasma data, and Kp index.
2. **ML Inference:** The `predictor.py` script routes this external telemetry through an XGBoost model to calculate the current aurora probability.
3. **API Serving:** The calculated parameters (aurora_score, geomagnetic_storm, etc.) are structured via Pydantic and served through the FastAPI endpoints (`/api/weather/forecast/global` and `/api/weather/telemetry/history`).
4. **Client Rendering:** The Next.js frontend fetches JSON data from the FastAPI endpoints and updates the React state, rendering immersive 3D globes and charts.
